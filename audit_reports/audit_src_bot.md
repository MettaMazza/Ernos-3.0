# Audit Report: `src/bot/` Subsystem

## Overview
The `src/bot/` directory is the core integration point with the Discord API. It defines the Discord client (`ErnosClient`), core bot global states, an AI-powered integrity auditor, and a massive suite of specialized Discord Cogs (`src/bot/cogs/`) that handle commands, events, and message orchestration. 

## Files in `src/bot/`

### 1. `client.py`
- **Function**: Defines `ErnosClient`, the primary `commands.Bot` subclass.
- **Key Logic**:
  - Initializes critical managers (`EngineManager`, `PromptManager`, `Hippocampus`, `Cerebrum`, Document parser tools) and attaches them to `self`.
  - Sets up the background scheduler and loads all cogs inside `setup_hook`.
  - Synchronizes Discord slash commands (`tree.sync()`).
  - Implements the Town Hall auto-save loop (`_auto_save_town_hall`).
- **Quote**:
  ```python
  class ErnosClient(commands.Bot):
      def __init__(self, *args, **kwargs):
          super().__init__(*args, **kwargs)
          self.engine_manager = EngineManager() # Load Model Provider
  ```

### 2. `globals.py`
- **Function**: Manages context-vars and global bot state to pass around the asynchronous architecture.
- **Key Logic**: Tracks `bot` instance, `active_message`, `active_channel`, `active_tracker` (for Live Status), and an `activity_log` for post-mortems.
- **Quote**:
  ```python
  active_message: ContextVar[Optional[discord.Message]] = ContextVar("active_message", default=None)
  ```

### 3. `integrity_auditor.py`
- **Function**: Acts as a safety net by performing LLM-based audits on Ernos's outbound responses just before they are sent.
- **Key Logic**: Uses `o1-mini` (or the fallback engine) to analyze the response against constraints (Sycophancy, Hallucination, Unwarranted Refusals, Cognitive Noise).
- **Quote**:
  ```python
  async def audit_response(user_message, bot_response, bot, user_id, conversation_context="", system_context="", tool_outputs=None):
      # ...
      response = await client.chat.completions.create(
          model="o1-mini",
          messages=[{"role": "user", "content": audit_prompt}]
      )
  ```

### 4. `post_mortem.py`
- **Function**: Analyzes failure states when the Cognitive Engine crashes and generates a detailed Markdown report (`generate_post_mortem`).
- **Key Logic**: Takes the global `activity_log`, `last_input`, and an `Exception`, generating a report that includes "What happened," "Where it broke," and a "Diagnosis."

## Files in `src/bot/cogs/`

The cogs directory contains 24 files logically split into commands, orchestration, and features.

### 5. Chat Engine Orchestration
This is arguably the most complex logic route in the entire Discord integration, split across multiple files to handle the message pipeline.
- `chat.py`: The main `ChatListener` (Orchestrator). Intercepts `on_message`, checks rate limits (`FluxCapacitor`), ignores bots, handles DM cooldowns. Calls preprocessing, triggers the `CognitionEngine`, and delegates response delivery.
- `chat_preprocessing.py`: Handles Phase 0 context building. Retrieves `early_hippocampus_recall` (Working memory, Graph, Lessons), fetches cross-channel context if mentioned, runs the `UnifiedPreProcessor` for triage, and injects interaction mode preferences (`professional` vs `default`).
- `chat_attachments.py`: Manages document injection, handles multi-turn `_image_cache` for context memory, performs provenance verification (to know if an image was generated by the bot or uploaded), and detects/blocks raw JSON backup copy-pastes.
- `chat_helpers.py`: Contains `AttachmentProcessor` (parsers for `.pdf`, `.docx`, `.csv`, `.epub`, etc. via various libraries like `pdfplumber`, `odfpy`, `olefile`) and `ReactionHandler` (for ingesting social signals from reacting to messages).
- `chat_response.py`: Finalizes Phase 4. Invokes the `integrity_auditor.py`, records to the `Hippocampus` (`observe`), resets the Visual Cortex lock, strips internal Cognitive Tape tags (`<EMIT>`), and dispatches the message natively to Discord (chunking if `> 2000` chars).

### 6. Admin Commands
These cogs are strictly protected and meant for server owners/devs. Use `settings.ADMIN_IDS` / `commands.is_owner()`.
- `admin.py`: General sysadmin commands `/clear`, `/silo`, `/devlog`, `/metrics`, `/kill`.
- `admin_engine.py`: `/engine`, `/model`, `/engines` commands to hot-swap LLMs and routing logic.
- `admin_lifecycle.py`: `/cycle_reset` (Hard reset bot state), `/nuke` (delete user memory), `/grounding_pulse` (System broadcast).
- `admin_moderation.py`: `/strike`, `/unstrike`, `/timeout_user` for managing trust metrics.
- `admin_reports.py`: `/status`, `/threats`, `/report` for security and telemetry data.
- `admin_testing.py`: `/testall` parses `MASTER_SYSTEM_TEST.md` and sequentially auto-executes the real `ToolRegistry` to ensure all backend tools function.
- `proxy_cog.py`: Enables the admin to send messages masquerading as Ernos using a DM forward, Discord message link paste, or `/proxy` command.

### 7. User Tools & Modifiers
- `mode_commands.py`: `/professional` and `/self` slash commands to toggle interaction styles in DMs (switches Ernos from terse/work mode to full philosophical interaction).
- `persona_commands.py`: `/persona`, `/persona_list`, `/persona_create`, `/persona_fork`. Allows users to switch to different AI characters in DMs or instantiate a temporary public persona thread in the Town Hall channel.
- `inbox_commands.py`: `/inbox` for users to check proactive, asynchronous messages that personas have queued up for them.
- `relationship_commands.py`: `/outreach_policy` and `/outreach_frequency`. Fine-grained controls so users define if/when the AI is allowed to initiate unprompted conversation (Outreach system hook).

### 8. Feature Specifics
- `mrn_commands.py`: "Mycelium Root Network" `/backup_my_shard` and `/restore_my_shard` (JSON Cryptographic Export/Import of local context), plus `/link_minecraft`.
- `monetization.py`: Synchronizes Discord Role changes (e.g. `pollinator`, `planter`, `terraformer`) with the backend `FluxCapacitor` rate-limiting tiers.
- `silo_commands.py`: `/leave` command to manually exit a managed private thread (Silo) if Discord UI is uncooperative.
- `skill_commands.py`: Admin/UI for `/approve_skill`, `/list_proposals`, `/schedule_skill` for the Python AST code injection tool. 
- `stop_command.py`: `/stop` sends an asyncio cancellation signal directly to the `CognitionEngine` to halt in-flight inference for that user.
- `support.py`: Support desk handler. Monitors the Support channel, opens a new thread, and triggers a modified Cognition pass with the `User Manual` injected directly into the system prompt and aggressive RAG recall to autonomously help users.
- `tape_commands.py`: `/restore_tape` manually rolls back a user's Cognitive Tape to a stable snapshot.
- `welcome.py`: Standard `on_member_join` listener but routed directly through the Cognitive Engine so the greeting is context-aware and personalized. Included Patreon links.

## Conclusion
The `src/bot/` subsystem is mature, heavily modularized, and extensively defensive. It demonstrates an advanced implementation of Discord.py with careful handling of asynchronous context and isolation of core logic (Cognition/PreProcessing/Attachments) away from basic I/O code.
