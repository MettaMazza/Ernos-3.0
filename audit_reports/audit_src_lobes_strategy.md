# Audit Report: `src/lobes/strategy/` The Strategic Center

## Overview
The "Strategy" lobe comprises the executive, metacognitive, and software engineering capabilities of Ernos. It evaluates scenarios, decomposes tasks, writes code, monitors system health, tunes its own prompt behavior, and polices user intent.

## 1. `architect.py` & `coder.py`
- **Function**: Software Engineering & Code Generation.
- **Key Logic**:
  - `ArchitectAbility` provides a high-level technical plan based on a user request.
  - `CoderAbility` operates an autonomous "Design -> Code -> Test -> Fix" execution loop.
  - Generates Python code, writes it to a temporary file, and runs it via `asyncio.create_subprocess_exec("python3")`.
  - Captures `stdout` and `stderr`. If execution fails (non-zero return code), the error output is fed back to the LLM for up to 10 retry/fix iterations.
  - Successfully tested code and a summary are broadcast to a Discord audit channel (`CODE_AUDIT_CHANNEL_ID`).

## 2. `gardener.py`
- **Function**: Knowledge Graph Maintenance.
- **Key Logic**:
  - Contains advanced Neo4j health scripts.
  - `connect_graph`: Uses Cypher to find nodes with low connectivity (`degree <= 2`). Selects candidate pairs using heuristics (shared neighbors, semantic overlap). Sends pairs to an LLM to hallucinate valid relationships (`RELATED_TO`, `PART_OF`), then validates them through the `OntologistAbility` before storing.
  - `refine_graph`: Identifies duplicate schema nodes via a custom Python Levenshtein string similarity function. A similarity > 0.95 triggers an auto-merge (APOC `apoc.refactor.mergeNodes` or manual relationship rewiring).

## 3. `goal.py` & `project.py`
- **Function**: Task and Goal Management.
- **Key Logic**:
  - `GoalAbility`: Scans user goal files (`goals.json`) to detect stagnation. Goals lacking an update for `> 3` days are flagged. Can use LLM to break high-level goals into 3-5 subtasks.
  - `ProjectLeadAbility`: Uses LLMs to decompose a project into milestones. Creates formal `Project` and `Milestone` nodes in the Neo4j Knowledge Graph connected by `HAS_MILESTONE` edges.

## 4. `introspection.py` & `performance.py`
- **Function**: System Metrics & Architectural Reflection.
- **Key Logic**:
  - `IntrospectionEngine`: Tracks internal software stats. Records Lobe call counts, response latency (P95), application layer memory sizes (by calculating the size of `data_dir` subdirectories), and error frequencies. Periodically checkpoints snapshots to analyze health trends and bottlenecks (e.g., "Slow response: >5000ms").
  - `PerformanceAbility`: Checks actual OS host metrics using `psutil` (CPU payload and Virtual Memory %).

## 5. `prompt_tuner.py` & `test_forge.py`
- **Function**: Self-Modification Systems.
- **Key Logic**:
  - Both act as "Staging" facilities. Ernos can analyze its own performance and propose changes, but cannot commit them directly.
  - `PromptTunerAbility`: Proposes changes to text files in `src/prompts/`. Operations include `replace`, `append`, `insert`, or `delete`. Modifying active system rules requires an Admin to manually approve the staged JSON proposal, which backs up the original and applies the mutation.
  - `TestForge`: Analyzes codebase components and generates comprehensive unit test scripts to untested code paths, saving them to `test_forge/staging`. An admin must promote them to `tests/`.

## 6. `predictor.py`
- **Function**: Mental Simulation.
- **Key Logic**:
  - Analyzes a proposed action ahead of execution to forecast outcomes. Outputs structured JSON detailing Confidence (0.0 - 1.0), Primary Risk, Secondary Risk, and a Recommendation (e.g., "Proceed with caution").

## 7. `sentinel.py`
- **Function**: Security & Threat Profiling.
- **Key Logic**:
  - Fast Static Pre-Filter: Deterministically blocks known injection prefixes (e.g., "[system emergency]").
  - Active Threat Assessor: Uses the active engine to score the user interaction out of 10 for both `threat` (toxicity, jailbreak) and `value` (creativity, intellectual depth).
  - Maintains an ongoing, moving-average threat profile per user over time. High threat interactions grant "strikes," while high value interactions redeem strikes.
  - Contains daily and "Master Cycle" loops to evaluate user behavioral drift over months.

## 8. `skill_forge.py`
- **Function**: Agent Skilling System.
- **Key Logic**:
  - Authoring mechanism where Ernos writes entirely new tool capabilities as `.md` files equipped with YAML frontmatter.
  - Supports `PRIVATE` (user-scoped) and `PUBLIC` (system-wide) execution scopes.
  - Ensures robust tamper-proofing utilizing SHA256 hashing (`approved_hash`) computed over the raw prompt instruction body.
  - Uses a `SAFE_TOOL_WHITELIST` to automatically approve `PRIVATE` skill integrations generated by autonomy layers, but strictly gates public/system mutations through an admin validation queue.
