# Audit Report: `src/backup/` Subsystem

## Overview
The `src/backup/` subsystem handles Ernos's data preservation, disaster recovery, and the "Mycelium Root Network" (MRN) user context exports. It allows users to download a shard of their personalized context and later restore it, allowing them to carry their memory across instances.

## Files in `src/backup/`

### 1. `manager.py`
- **Function**: Coordinator for all backup operations. Delegates logic to focused submodules (`export.py`, `restore.py`, `verify.py`). It also handles pure system-level directory backups (the daily 2 PM cron-job).
- **Key Logic**:
  - `daily_backup` creates a full copy of `/users` and `/core` using `shutil.copytree`.
  - Cleans up daily backups older than 7 days (`_cleanup_old_backups`).
- **Quote**:
  ```python
  class BackupManager:
      """
      Manages system backups and user context exports.
      - Daily 2pm backup: Full system backup (internal only)
      - Cycle reset export: User context sent via DM (once per 24hrs)
      - Retention: Only last 7 days of daily backups kept
      """
  ```

### 2. `export.py`
- **Function**: Handles the logic for creating backup shards containing context, traces, the knowledge graph, and public timelines.
- **Key Logic**: 
  - Iterates through the user's silo to package all files into a JSON structure (`export_user_context`).
  - Implements a strict 24-hour rate limit on exports via `rate_limits.json` to prevent spam/abuse.
  - Queries Neo4j (`KnowledgeGraph`) to dump all `labels` and `properties` associated with the user.
  - Generates a checksum for the export so it can be verified later.
- **Quote**:
  ```python
  if last and (datetime.now() - last).total_seconds() < self.EXPORT_COOLDOWN_HOURS * 3600:
      logger.info(f"User {user_id} export rate limited")
      return None
  ```

### 3. `verify.py`
- **Function**: Cryptographic verification of MRN shards.
- **Key Logic**: Uses `HMAC-SHA256` combined with a local, securely baked-in salt (from `ProvenanceManager`) to sign exports. Upon restore, it verifies the checksum to ensure the file was not tampered with and was actually generated by a compatible Ernos instance.
- **Quote**:
  ```python
  payload = f"{self._salt}:{user_id}:{exported_at}:{context_str}"
  return hashlib.sha256(payload.encode("utf-8")).hexdigest()
  ```

### 4. `restore.py`
- **Function**: Rehydrates a user's context from an uploaded shard.
- **Key Logic**: 
  - Step 1: verifies authenticity.
  - Step 2: writes raw text files back to the user's silo and the public timeline.
  - Step 3: reconstitutes `reasoning.log` from traces.
  - Step 4: reconstructs the in-memory state: replays `.jsonl` logs into `hippocampus.working` and re-embeds text chunks into the VectorStore.
  - Step 5: re-injects the exported Neo4j nodes back into the `KnowledgeGraph`.
- **Vulnerability/Robustness Note**: The restorer contains fallback mechanisms for legacy formats and ensures files missing required headers are ignored. However, heavy restores could block the event loop temporarily.
- **Quote**:
  ```python
  # Re-embed context into VectorStore
  # ...
  chunk = content[:5000]
  embedding = hippocampus.embedder.get_embedding(chunk)
  if embedding:
      hippocampus.vector_store.add_element(...)
  ```

## Conclusion
The backup system provides cryptographic authenticity for user-held context (MRN shards) and comprehensive daily snapshots. It delegates duties correctly to prevent a monolith, mapping out strict rate limits and multi-layer state restoration (Raw files, SQLite, ChromaDB, Neo4j).
