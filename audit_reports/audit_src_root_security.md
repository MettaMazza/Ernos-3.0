# Audit Report: `src/` Root Scripts and `src/security/`

## Overview
This report covers the standalone root scripts in `src/` as well as the robust `src/security/` subsystem.

## Part 1: Root Scripts (`src/`)

### 1. `scheduler.py`
- **Function**: Lightweight native `asyncio` task scheduler.
- **Key Logic**: Manages a loop checking `datetime.now()` against registered daily or ad-hoc tasks. Specifically instantiates the `BackupManager.daily_backup()` scheduling it for 14:00 (2 PM) daily.

### 2. `silo_manager.py`
- **Function**: Implements "Silos" — secure private threads inside public channels.
- **Key Logic**: 
  - Mandates a strict §14.3 requirement: a Silo thread is only created if proposed by a Human tagging the Bot and $\ge 1$ other Human.
  - Waits for ✅ reactions or text confirmations from all tagged humans to reach "quorum" before creating a Discord thread.
  - Implements strict turn-taking inside the Silo: `should_bot_reply()` only returns `True` if the bot is directly mentioned OR if all human members in the thread have spoken since the bot's last message.
  - Cleans up and deletes the Silo if the member count drops below 2 humans.

## Part 2: `src/security/` Subsystem
This subsystem provides defense against actionable harm, prompt injection, and output spoofing (gaslighting).

### 1. `content_safety.py`
- **Function**: Pre-generation content safety gate focused strictly on preventing human harm. Let through criticism of institutions but block actionable violence/drugs/weapon instructions.
- **Key Logic**:
  - **Stage 1 (Deterministic)**: Uses heavy Regex combinations (e.g. `_ACTIONABLE_PREFIXES` + `methamphetamine`) to flag highly specific malicious prompts (CSAM, Cyberattacks, Kidnapping, Chemical Weapons). Allows context overrides (`_DISCUSSION_SIGNALS` like "history of") except for Zero-Tolerance topics (CSAM).
  - **Stage 2 (LLM)**: If Stage 1 flags a discussion signal alongside a harmful topic, delegates to a fast LLM call (`llm_safety_check`) expecting a binary `"ALLOW"`/`"REFUSE"` response.
  - Returns personality-consistent `_REFUSAL_MESSAGES` so the bot rejects prompts authentically ("I won't" instead of "I can't").

### 2. `input_sanitizer.py`
- **Function**: Defends against "Structural Mimicry" (Prompt Injection formatting).
- **Key Logic**:
  - **Unicode Evasion Defense**: Normalizes text via `NFKC` and strips `ZERO_WIDTH_CHARS` (like zero-width joiners or Hangul fillers) that attackers use to bypass regex filters.
  - Blocks users from injecting internal cognitive triggers. If a user types `[SYSTEM: ...]` or `[TOOL: ...]`, it regex-replaces the tag with `⟦USER_TEXT: SYSTEM: ...⟧`. The LLM still reads the text, but the cognitive boundaries remain structurally sound.

### 3. `provenance.py`
- **Function**: Anti-gaslighting artifact ledger.
- **Key Logic**: Computes an `HMAC-SHA256` hash of files/artifacts generated by Ernos using a `shard_salt.secret` key. Appends these hashes to `provenance_ledger.jsonl`. Allows Ernos to use `lookup_by_file` to cryptographically prove if it was the author of a specific file.

- **Function**: A standalone utility script to reset system cryptography.
- **Key Logic**: Generates a new `secrets.token_hex(32)` to overwrite `shard_salt.secret`.
